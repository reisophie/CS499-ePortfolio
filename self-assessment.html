Professional Self-Assessment

Completing the Computer Science program and building this ePortfolio has helped me see how much I have grown, not only in writing code, but also in how I think about software quality, organization, and long-term improvement. Earlier in the program, I was mainly focused on making projects function correctly. As I moved through my coursework, I became better at looking at a project more critically and asking whether the design made sense, whether the data was handled well, and whether the solution would still be maintainable later. This capstone gave me the chance to revisit earlier work and improve it with a more professional mindset.

One of the most important things I gained through the program was a better understanding of how the major areas of computer science connect. In this portfolio, I used one artifact, my CS 360 Daily Weight Tracker application, across software design and engineering, algorithms and data structures, and databases. That choice helped me show that these areas are not separate in real development. A user-facing application depends on design decisions, structured logic, and reliable data handling all at once. Working through each enhancement made me more confident in my ability to evaluate an existing system, identify weaknesses, and improve it in a meaningful way.

Throughout the program, I also strengthened my ability to communicate technical work more clearly. That includes writing narratives, explaining design decisions, reflecting on trade-offs, and presenting my work in a way that is understandable to different audiences. I learned that strong communication matters just as much as technical ability because software development is rarely done in isolation. Developers need to explain ideas, justify decisions, and communicate progress clearly.

This program also helped me better understand collaboration and stakeholder needs. Even when a class project is completed individually, there is still an expectation that the work solves a problem, meets requirements, and reflects user needs. In my weight-tracking app, I had to think about how a user would interact with the application, how validation should guide the experience, and how the structure of the code would affect future improvements. That mindset connects directly to working with stakeholders in professional environments, where a successful solution has to balance technical choices with practical needs.

Another area of growth for me has been security and dependability. As I moved through the program, I became more interested in cybersecurity and more aware that security should not be treated as an afterthought. In this portfolio, that growth is visible in the way I improved validation, reduced weak data handling, and replaced plain-text password storage with hashed values in the database enhancement. Even though this was still a class project, making those improvements showed that I am thinking more carefully about privacy, reliability, and the risks that come from poor design decisions.

This ePortfolio demonstrates how I have met the program outcomes. I have shown that I can use collaborative and communication strategies by presenting a code review, written narratives, and a professional portfolio that explains my work clearly. I have shown that I can design and evaluate computing solutions by improving an existing application through better structure, validation, logic, and data organization. I have shown that I can use appropriate tools and techniques to create practical computing solutions, including Java, Android Studio, SQLite, RecyclerView-based interfaces, helper classes, repositories, and session management. Finally, I have shown a stronger security mindset by improving validation, strengthening data integrity, and making better decisions around password storage and user-session handling.

Overall, this capstone reflects my growth as a computer science student and the direction I want to continue developing professionally. I am especially interested in roles that involve improving systems, supporting operations, and making technology more reliable and secure. This portfolio is important to me because it shows more than finished assignments. It shows that I can evaluate my own work, respond to weaknesses, and make thoughtful improvements that better reflect my current skills.
